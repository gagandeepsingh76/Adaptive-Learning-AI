# API

The backend exposes JSON APIs from the FastAPI application in `backend/app/main.py`.

## Core Endpoints

| Method | Path | Status | Purpose |
| --- | --- | ---: | --- |
| `GET` | `/health` | 200 | Process health and release metadata |
| `GET` | `/health/live` | 200 | Lightweight liveness probe |
| `GET` | `/health/ready` | 200 | Database-backed readiness probe |
| `POST` | `/roadmap` | 201 | Generate, persist, chunk, embed, and index a roadmap |
| `POST` | `/project` | 201 | Generate a portfolio project from a roadmap or direct skill list |
| `POST` | `/chat` | 200 | Answer a roadmap-grounded question with RAG citations |
| `GET` | `/progress/{roadmap_id}` | 200 | Read derived progress summary |
| `PATCH` | `/progress/{roadmap_id}` | 200 | Update progress for a roadmap hierarchy target |
| `GET` | `/openapi.json` | 200 | OpenAPI schema |
| `GET` | `/docs` | 200 | FastAPI Swagger UI |

## Request Rules

- JSON bodies use strict Pydantic schemas.
- Unknown fields are rejected.
- UUID path and body fields are validated.
- Payload size is limited by `ALA_MAX_REQUEST_BYTES`.
- Learner scope is supplied by `X-Learner-ID` unless local anonymous mode is enabled.

## Error Shape

Errors are returned through a centralized envelope:

```json
{
  "error": {
    "code": "REQUEST_VALIDATION_ERROR",
    "message": "The request payload is invalid.",
    "request_id": "request-id",
    "retryable": false
  }
}
```

Validation errors include safe field details. Unexpected exceptions return a generic
`INTERNAL_ERROR` message without stack traces.

## OpenAPI

OpenAPI is generated by FastAPI from the implemented schemas and route metadata. Release verification
checks that `/roadmap`, `/project`, and `/chat` are present in `/openapi.json`.
